# 서브쿼리 (Subquery)

> *(SQL 예제는 모두 **MySQL** 기준)*

**서브쿼리**: SQL문을 실행하는 데 필요한 데이터를 추가로 조회하기 위해, **SQL문 내부에서 사용하는 SELECT문**을 의미한다.

- 괄호 `( )` 안에 들어가는 SELECT문이 **서브쿼리**, 그것을 감싸는 바깥 쿼리가 **메인 쿼리**.
- 서브쿼리가 **먼저** 실행되어 값을 만들고, 그 값을 메인 쿼리가 사용한다.

### 서브쿼리의 위치별 분류

| 위치 | 이름 | 역할 |
| --- | --- | --- |
| **WHERE** 절 | 조건 서브쿼리 | 조회 조건의 비교 값으로 사용 |
| **FROM** 절 | 인라인 뷰(Inline View) | 서브쿼리 결과를 하나의 임시 테이블처럼 사용 |
| **SELECT** 절 | 스칼라 서브쿼리 | 열 자리에서 값 하나를 가져옴 |

## WHERE 절에서의 서브쿼리

### 단계별로 풀어보기 — "JONES보다 급여가 높은 사원"

```sql
-- 1. 사원 이름이 JONES인 사원의 급여 출력
SELECT SAL
FROM EMP
WHERE ENAME = 'JONES';
```

```sql
-- 2. 급여가 2975보다 높은 사원 정보 출력
SELECT ENAME
FROM EMP
WHERE SAL > 2975;
```

```sql
-- 1 + 2. JONES의 급여보다 높은 급여를 받는 사원 출력
--   → 2975라는 값을 직접 쓰는 대신, 1번 쿼리를 서브쿼리로 넣는다
SELECT *
FROM EMP
WHERE SAL > (SELECT SAL FROM EMP WHERE ENAME = 'JONES');
```

> 💡 서브쿼리를 쓰면 `2975` 같은 값을 미리 알지 못해도, 그 값을 구하는 쿼리를 그대로 끼워 넣을 수 있다.

### 연습 문제

<details>
<summary><b>실습)</b> 'ALLEN'의 직무와 같은 사람의 이름, 부서명, 급여, 직무를 출력하시오.</summary>

```sql
SELECT E.ENAME, D.DNAME, E.SAL, E.JOB
FROM EMP E, DEPT D
WHERE E.DEPTNO = D.DEPTNO
  AND E.JOB = (SELECT JOB FROM EMP WHERE ENAME = 'ALLEN');
```

</details>

<details>
<summary><b>실습)</b> 전체 사원의 평균 임금보다 많이 받는 사원의 사원번호, 이름, 부서명, 입사일, 지역, 급여를 출력하시오.</summary>

```sql
SELECT E.EMPNO, E.ENAME, D.DNAME, E.HIREDATE, D.LOC, E.SAL
FROM EMP E, DEPT D
WHERE E.DEPTNO = D.DEPTNO
  AND E.SAL > (SELECT AVG(SAL) FROM EMP);
```

</details>

<details>
<summary><b>실습)</b> EMP·DEPT 테이블에서 부서번호가 20이고, 부서번호가 일치하며, EMP 평균 급여보다 높은 직원을 출력하시오.</summary>

```sql
SELECT *
FROM EMP E, DEPT D
WHERE E.DEPTNO = D.DEPTNO
  AND E.DEPTNO = 20
  AND E.SAL > (SELECT AVG(SAL) FROM EMP);
```

</details>

<details>
<summary><b>실습)</b> 각 부서별 최고 급여와 동일한 급여를 받는 사원 정보를 출력하시오.</summary>

```sql
SELECT *
FROM EMP
WHERE SAL IN (SELECT MAX(SAL) FROM EMP GROUP BY DEPTNO);
```

> 💡 서브쿼리 결과가 **여러 행**(부서별 최고 급여 여러 개)이므로 `=`이 아닌 **`IN`** 을 사용한다.

</details>

## 단일행 vs 다중행 서브쿼리

서브쿼리가 돌려주는 **행의 개수**에 따라 사용할 연산자가 다르다.

| 구분 | 결과 행 수 | 사용 연산자 |
| --- | --- | --- |
| **단일행 서브쿼리** | 1개 | `=`, `>`, `<`, `>=`, `<=`, `<>` |
| **다중행 서브쿼리** | 여러 개 | `IN`, `NOT IN`, `ANY`, `ALL`, `EXISTS` |

> ⚠️ 서브쿼리가 여러 행을 돌려주는데 `=` 같은 단일행 연산자를 쓰면 오류가 난다. 이럴 땐 `IN` 등을 사용한다.

| 다중행 연산자 | 의미 |
| --- | --- |
| `IN` | 서브쿼리 결과 중 **하나라도 일치**하면 참 |
| `ANY` | 서브쿼리 결과 중 **어느 하나라도** 조건을 만족하면 참 (`> ANY` = 최솟값보다 크면) |
| `ALL` | 서브쿼리 결과 **모두**에 대해 조건을 만족해야 참 (`> ALL` = 최댓값보다 크면) |
| `EXISTS` | 서브쿼리 결과가 **한 행이라도 존재**하면 참 |

## FROM 절에서의 서브쿼리 (인라인 뷰)

서브쿼리 결과를 **하나의 임시 테이블처럼** 사용한다. 반드시 별칭을 붙인다.

```sql
SELECT E10.ENAME, D.DNAME
FROM (SELECT * FROM EMP WHERE DEPTNO = 10) E10,
     (SELECT * FROM DEPT) D
WHERE E10.DEPTNO = D.DEPTNO;
```

> 💡 `(SELECT ... )`에 붙인 별칭(`E10`, `D`)이 임시 테이블 이름 역할을 한다. 큰 테이블에서 필요한 부분만 미리 추려 쓸 때 유용하다.

---

## SELECT 절에서의 서브쿼리 (스칼라 서브쿼리)

열을 출력하는 자리에서, **값 하나(스칼라)** 를 가져온다.

```sql
SELECT EMPNO, ENAME, JOB, SAL,
       (SELECT GRADE FROM SALGRADE
        WHERE E.SAL BETWEEN LOSAL AND HISAL) AS SALGRADE,
       DEPTNO,
       (SELECT DNAME FROM DEPT
        WHERE E.DEPTNO = DEPT.DEPTNO) AS DEPT
FROM EMP E;
```

> ⚠️ 스칼라 서브쿼리는 행마다 **값 1개**만 돌려줘야 한다. 여러 행이 나오면 오류가 발생한다.
