# MySQL InnoDB 정리

> MySQL의 **기본 저장 엔진(storage engine)** 인 InnoDB가 무엇이고, 왜 쓰는지 정리한 문서입니다.

---

## 1. 저장 엔진이란?

MySQL은 구조가 두 층으로 나뉘어 있습니다.

- **위층(SQL 층)**: SQL을 해석하고 최적화한다. (쿼리 파싱, 실행 계획 등)
- **아래층(저장 엔진)**: 데이터를 실제로 디스크에 저장·관리하고, 잠그고, 복구한다.

MySQL의 특징은 이 저장 엔진을 **테이블마다 바꿔 끼울 수 있다**는 점입니다(InnoDB, MyISAM, MEMORY 등). 그중 **InnoDB가 MySQL 5.5(2010)부터 기본값**이며, 사실상 표준입니다.

> InnoDB = "데이터를 실제로 읽고 쓰고 잠그고 복구하는 부품"

---

## 2. InnoDB의 핵심 특징

### (1) 트랜잭션 (ACID)
`COMMIT` / `ROLLBACK`을 지원합니다. 여러 작업을 하나로 묶어 **전부 성공 아니면 전부 취소**할 수 있습니다.

| ACID | 의미 |
|------|------|
| **A**tomicity (원자성) | 전부 실행되거나, 전부 취소되거나 |
| **C**onsistency (일관성) | 규칙에 맞는 상태만 유지 |
| **I**solation (격리성) | 동시에 실행돼도 서로 간섭하지 않음 |
| **D**urability (지속성) | 커밋된 데이터는 장애가 나도 남음 |

### (2) 행 단위 잠금 (Row-level Locking)
한 명이 어떤 행을 고치는 동안, 다른 사람은 **다른 행**을 동시에 고칠 수 있습니다. → 동시성이 좋습니다.
(옛 MyISAM은 **테이블 전체**를 잠가서, 한 명이 쓰면 나머지가 모두 기다려야 했습니다.)

### (3) 외래키 (Foreign Key)
"이 글의 `user_id`는 반드시 `user` 테이블에 있어야 한다" 같은 **참조 무결성**을 DB가 강제합니다.

### (4) 장애 복구
변경을 먼저 **redo 로그**에 기록(write-ahead)해두기 때문에, 서버가 갑자기 죽어도 다시 켜면 복구할 수 있습니다.

### (5) MVCC (다중 버전 동시성 제어)
읽기가 쓰기를 막지 않습니다. 읽는 쪽은 그 시점의 **일관된 스냅샷**을 보고, 쓰는 쪽은 따로 작업합니다.

### (6) 클러스터형 인덱스
데이터를 **기본 키(PK) 순서대로 물리적으로 정렬**해 저장합니다.
- PK로 찾는 게 매우 빠릅니다.
- 그래서 **PK 설계가 성능에 직접 영향**을 줍니다.
- 메모리에 데이터를 캐시하는 **버퍼 풀(buffer pool)** 도 InnoDB 성능의 핵심입니다.

---

## 3. InnoDB vs MyISAM (옛 기본 엔진)

| 항목 | InnoDB | MyISAM |
|------|--------|--------|
| 트랜잭션 | 지원 ✅ | 없음 ❌ |
| 잠금 단위 | 행(row) | 테이블 전체 |
| 외래키 | 지원 ✅ | 없음 ❌ |
| 장애 복구 | 강함(redo log) | 약함 |
| 동시 쓰기 | 좋음 | 나쁨 |
| MVCC | 지원 | 없음 |
| 현재 위상 | **기본 / 권장** | 레거시 |

> 결론: 특별한 이유가 없으면 **InnoDB**를 씁니다. (그리고 `CREATE TABLE`의 기본값이라 자동으로 InnoDB로 만들어집니다.)

---

## 4. 실제 프로젝트와의 연결

회원 관리·게시판 프로젝트에서 쓰는 기능들이 전부 InnoDB 덕분입니다.

- **외래키**: `content.user_id` → `user(user_id)` 참조
- **트랜잭션**: 회원 탈퇴 시 "회원 + 그 회원의 글"을 `commit`/`rollback`으로 한 번에 처리
- **동시성**: 여러 사용자가 동시에 글을 써도 행 단위 잠금으로 안전

만약 테이블을 MyISAM으로 만들었다면 외래키는 무시되고 트랜잭션도 되지 않았을 것입니다.

---

## 5. 확인 방법

```sql
-- 특정 테이블의 엔진 확인
SHOW CREATE TABLE 테이블명;     -- 끝에 ENGINE=InnoDB 표시

-- 전체 테이블 상태(엔진 포함)
SHOW TABLE STATUS;

-- 테이블을 만들 때 엔진 명시(보통은 생략해도 InnoDB)
CREATE TABLE member (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50)
) ENGINE=InnoDB;
```

---

## 6. 한 줄 요약

> **InnoDB = 트랜잭션 + 행 잠금 + 외래키 + 장애 복구를 갖춘 MySQL의 기본 저장 엔진.**
> 안전하고 동시성이 좋아, 특별한 이유가 없으면 항상 이걸 쓴다.
