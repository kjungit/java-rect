# 집합 연산자

> *(SQL 예제는 모두 **MySQL** 기준)*

두 개 이상의 SELECT문 결과를 하나로 묶는 연산자다.

| 연산자 | 의미 | 중복 처리 | MySQL 지원 |
| --- | --- | --- | --- |
| **UNION** | 합집합 | 중복 제거 | ○ |
| **UNION ALL** | 합집합 | 중복 포함(모두 출력) | ○ |
| **INTERSECT** | 교집합 | 중복 제거 | ○ (MySQL 8.0.31+) |
| **EXCEPT** | 차집합 | 중복 제거 | ○ (MySQL 8.0.31+) |

> ⚠️ **공통 규칙**: 연결되는 SELECT문은 **열의 개수**가 같아야 하고(다르면 오류), **각 열의 자료형(순서별)**도 맞춰야 한다. 단, MySQL은 자료형이 달라도 오류 없이 암묵적으로 형변환해 실행하므로 의미가 왜곡될 수 있다.

---

## UNION : 합집합

연결된 SELECT문의 결과를 합집합으로 묶어주며, **결과 값의 중복은 제거**된다.

```sql
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10
UNION
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 20;
```

### 잘못된 예 — 열의 수가 다를 때 (오류)

```sql
SELECT EMPNO, ENAME, SAL          -- 열 3개
FROM EMP
WHERE DEPTNO = 10
UNION
SELECT EMPNO, ENAME, SAL, DEPTNO  -- 열 4개 → 오류
FROM EMP
WHERE DEPTNO = 20;
```

### 주의 — 자료형이 다를 때 (MySQL은 오류 없이 실행됨)

열 개수만 같으면, 순서별 자료형이 달라도 **MySQL은 오류를 내지 않고** 암묵적으로 형변환해 실행한다. (※ **Oracle**은 이 경우 오류)

```sql
SELECT DEPTNO, EMPNO, ENAME, SAL
FROM EMP
WHERE DEPTNO = 10
UNION
SELECT EMPNO, ENAME, SAL, DEPTNO  -- 순서별 자료형 불일치 → MySQL은 실행(형변환), Oracle은 오류
FROM EMP
WHERE DEPTNO = 20;
```

> ✅ **열의 수**는 반드시 같아야 하며(다르면 오류), **자료형**도 맞추지 않으면 형변환으로 값이 왜곡될 수 있다.

### 주의 — 열 의미가 맞지 않아도 자료형만 같으면 실행됨

출력은 되지만 **데이터의 의미가 섞여 신뢰도가 낮다.**

```sql
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10
UNION
SELECT SAL, JOB, DEPTNO, SAL      -- 자료형은 맞지만 의미가 엉킴
FROM EMP
WHERE DEPTNO = 20;
```

---

## UNION ALL : 합집합 (중복 포함)

연결된 SELECT문의 결과를 합집합으로 묶되, **중복된 결과도 제거하지 않고 모두 출력**한다.

```sql
-- UNION : 중복 제거 → 한 번만 출력
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10
UNION
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10;
```

```sql
-- UNION ALL : 중복 포함 → 두 번 모두 출력
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10
UNION ALL
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10;
```

---

## INTERSECT : 교집합

앞 SELECT문의 결과 중, 뒤 SELECT문의 결과와 **공통으로 존재하는 데이터만** 출력한다.

```sql
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
INTERSECT
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10;
```

> ℹ️ `INTERSECT`는 **MySQL 8.0.31 이상**에서 지원한다. 그 이전 버전에서는 `INNER JOIN`이나 `IN` / `EXISTS`로 교집합을 구현해야 한다.

```sql
-- INTERSECT 미지원 버전에서의 대체 (IN 사용 예)
-- 위 INTERSECT(전체 ∩ DEPTNO=10)와 동일한 결과
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE EMPNO IN (SELECT EMPNO FROM EMP WHERE DEPTNO = 10);
```

---

## EXCEPT : 차집합

앞 SELECT문의 결과에서, 뒤 SELECT문의 결과에 있는 데이터를 **빼고** 출력한다.

```sql
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
EXCEPT
SELECT EMPNO, ENAME, SAL, DEPTNO
FROM EMP
WHERE DEPTNO = 10;
```

> ℹ️ 차집합 연산자 이름은 DBMS마다 다르다.
> - **MySQL**(8.0.31+) / 표준 SQL: `EXCEPT`
> - **Oracle**: `MINUS`
>
> 참고로 `UNION` / `UNION ALL` / `INTERSECT`는 MySQL과 Oracle에서 이름이 동일하다.
