# 트랜잭션(Transaction) 정리

> 여러 작업을 **하나의 단위로 묶어, 전부 성공하거나 전부 취소**되게 하는 트랜잭션을 정리한 문서입니다.

---

## 1. 트랜잭션이란?

**트랜잭션 = 여러 SQL 작업을 하나의 묶음으로 다루는 것.**
그 묶음은 **전부 반영(commit)되거나, 전부 취소(rollback)** 됩니다. 중간까지만 반영되는 일은 없습니다.

> 한 줄 요약: "**모두 성공 아니면 모두 실패(All or Nothing)**"

---

## 2. 왜 필요한가?

### 예시 ① 계좌 이체
A에서 1만원을 빼고 → B에 1만원을 넣는다. 이 둘은 **항상 같이** 일어나야 합니다.
만약 "A에서 빼기"는 성공했는데 "B에 넣기"가 실패하면? → 돈이 **공중분해**됩니다.
트랜잭션으로 묶으면, 하나라도 실패할 때 **전부 되돌려서** 이런 사고를 막습니다.

### 예시 ② 회원 탈퇴 (게시판 프로젝트)
회원을 지울 때 → 그 회원이 쓴 글도 같이 지워야 합니다.
"회원 삭제"는 됐는데 "글 삭제"가 실패하면 → **주인 없는 글**이 남습니다.
이 둘을 트랜잭션으로 묶으면, 하나라도 실패 시 둘 다 취소됩니다.

---

## 3. ACID — 트랜잭션이 보장하는 4가지

| 글자 | 이름 | 의미 |
|------|------|------|
| **A** | Atomicity (원자성) | 전부 실행되거나, 전부 취소되거나 |
| **C** | Consistency (일관성) | 규칙에 맞는 상태만 유지 |
| **I** | Isolation (격리성) | 동시에 실행돼도 서로 간섭하지 않음 |
| **D** | Durability (지속성) | 커밋된 데이터는 장애가 나도 남음 |

---

## 4. COMMIT / ROLLBACK 과 autocommit

- **COMMIT**: 지금까지의 작업을 **확정**(저장)한다.
- **ROLLBACK**: 지금까지의 작업을 **전부 취소**한다(트랜잭션 시작 시점으로 되돌림).

> ⚠️ **autocommit 주의**: MySQL은 기본이 **autocommit = ON**입니다. 즉 SQL 한 줄을 실행할 때마다 **자동으로 COMMIT**됩니다. 그래서 여러 작업을 한 트랜잭션으로 묶으려면, **명시적으로 트랜잭션을 시작**하거나 autocommit을 꺼야 합니다.

---

## 5. SQL에서 트랜잭션 쓰기

```sql
START TRANSACTION;          -- 트랜잭션 시작 (BEGIN 도 가능)

DELETE FROM content WHERE user_id = 'hong';   -- ① 글 삭제
DELETE FROM user    WHERE user_id = 'hong';   -- ② 회원 삭제

COMMIT;     -- 둘 다 성공 → 확정
-- ROLLBACK;  -- 문제가 있으면 → 전부 취소
```

`START TRANSACTION`과 `COMMIT` 사이의 작업들이 **하나의 묶음**이 됩니다.

---

## 6. JDBC에서 트랜잭션 쓰기 ⭐

자바에서는 `Connection`으로 트랜잭션을 제어합니다. 핵심은 세 가지예요.

```java
Connection conn = null;
try {
    conn = getConnection();
    conn.setAutoCommit(false);          // ① 트랜잭션 시작 (자동 커밋 끄기)

    // 같은 conn으로 여러 작업 수행
    deleteContentByUser(conn, userId);  // ② 글 삭제
    deleteUser(conn, userId);           // ③ 회원 삭제

    conn.commit();                      // ④ 둘 다 성공 → 확정
} catch (SQLException e) {
    if (conn != null) conn.rollback();  // ⑤ 하나라도 실패 → 전부 취소
    throw new RuntimeException(e);
} finally {
    if (conn != null) {
        try {
            conn.setAutoCommit(true);   // 원래대로 복구
            conn.close();
        } catch (SQLException ignored) {}
    }
}
```

| 단계 | 코드 | 의미 |
|------|------|------|
| 시작 | `setAutoCommit(false)` | 여기부터 한 트랜잭션 |
| 확정 | `commit()` | 모두 성공 시 저장 |
| 취소 | `rollback()` | 실패 시 전부 되돌림 |

> ⚠️ **가장 중요한 함정**: 트랜잭션으로 묶을 작업들은 **반드시 같은 `Connection`** 을 써야 합니다.
> 메서드마다 `getConnection()`으로 **새 연결을 열면**, 각각이 **별개의 트랜잭션**이 되어 묶이지 않아요.
> → 그래서 트랜잭션이 필요한 작업은 `Connection`을 **인자로 넘겨** 공유시킵니다.

## 7. 격리 수준 (Isolation Level)

여러 트랜잭션이 **동시에** 돌 때, 서로 얼마나 간섭을 허용할지 정하는 단계입니다. 느슨할수록 빠르지만 문제가 생기고, 엄격할수록 안전하지만 느려요.

### 동시성에서 생기는 문제
- **Dirty Read(오손 읽기)**: 아직 **커밋 안 된** 남의 데이터를 읽음
- **Non-Repeatable Read(반복 불가)**: 같은 행을 두 번 읽었는데 값이 **다름**(사이에 누가 수정·커밋)
- **Phantom Read(팬텀)**: 같은 조건으로 두 번 조회했는데 **행 개수가 다름**(사이에 삽입·삭제)

### 4가지 격리 수준
| 수준 | Dirty | Non-Repeatable | Phantom |
|------|:---:|:---:|:---:|
| READ UNCOMMITTED | 발생 | 발생 | 발생 |
| READ COMMITTED | 막음 | 발생 | 발생 |
| **REPEATABLE READ** (MySQL 기본) | 막음 | 막음 | 거의 막음* |
| SERIALIZABLE | 막음 | 막음 | 막음 |

> *MySQL InnoDB의 기본은 **REPEATABLE READ**이고, 갭 락(gap lock)으로 팬텀까지 상당 부분 막아줍니다.

## 8. 실제 프로젝트와의 연결

게시판의 **회원 탈퇴**가 트랜잭션의 대표 예시입니다.

```
회원 탈퇴 = ① 그 회원의 글 삭제  +  ② 회원 삭제
→ 둘을 한 트랜잭션으로 묶어, 하나라도 실패하면 둘 다 취소
```

이렇게 하면 "글은 지워졌는데 회원은 남거나", 그 반대인 **중간 상태**가 생기지 않습니다.
(이런 트랜잭션·롤백이 가능한 이유는 테이블이 **InnoDB 엔진**이기 때문입니다. MyISAM이면 안 됩니다.)

---

## 9. 한 줄 요약

> **트랜잭션 = 여러 작업을 하나로 묶어 "모두 성공 아니면 모두 취소"를 보장하는 것.**
> SQL은 `START TRANSACTION ~ COMMIT/ROLLBACK`, 자바는 `setAutoCommit(false) → commit()/rollback()`.
> 단, **같은 Connection**을 써야 한 트랜잭션으로 묶인다.
