# DML (데이터 조작어)

> *(SQL 예제는 모두 **MySQL** 기준)*

데이터를 **추가·수정·삭제**하는 명령어를 **DML**(Data Manipulation Language)이라고 한다.

| 명령어      | 역할     |     |
| -------- | ------ | --- |
| `INSERT` | 데이터 추가 |     |
| `UPDATE` | 데이터 수정 |     |
| `DELETE` | 데이터 삭제 |     |

---

## INSERT — 데이터 추가

### 형식

```sql
INSERT INTO 테이블명 [(열1, 열2, ..., 열N)]
VALUES (열1데이터, 열2데이터, ..., 열N데이터);
```

### 실습용 테이블 만들기

```sql
-- DEPT 테이블의 구조와 데이터를 그대로 복사
CREATE TABLE DEPT_TEMP AS SELECT * FROM DEPT;
```

### 여러 가지 INSERT 방법

```sql
-- 1) 열 지정 삽입
INSERT INTO DEPT_TEMP (DEPTNO, DNAME, LOC)
VALUES (50, 'DATABASE', 'SEOUL');
```

```sql
-- 2) 열 지정 생략 (모든 열에 순서대로 값을 넣을 때만 가능)
INSERT INTO DEPT_TEMP
VALUES (60, 'NETWORK', 'BUSAN');
```

```sql
-- 3) NULL 명시적 삽입
INSERT INTO DEPT_TEMP (DEPTNO, LOC, DNAME)
VALUES (70, NULL, 'WEB');
```

```sql
-- 4) 빈 문자열 삽입 ('' → 길이 0인 문자열, NULL과는 다름)
INSERT INTO DEPT_TEMP (DEPTNO, DNAME, LOC)
VALUES (80, 'MOBILE', '');
```

```sql
-- 5) NULL 암시적 삽입 (값을 안 준 열은 자동으로 NULL)
INSERT INTO DEPT_TEMP (DEPTNO, LOC)
VALUES (90, 'INCHEON');
```

### 구조만 복사하기 (데이터 없이)

```sql
CREATE TABLE EMP_TEMP AS SELECT * FROM EMP WHERE 1 <> 1;
```

> 💡 `WHERE 1 <> 1`은 항상 **거짓(FALSE)** 이므로 어떤 행도 조건을 통과하지 못한다. → **열 구조만** 복사되고 데이터는 들어가지 않는다.

### 서브쿼리로 여러 데이터 한 번에 추가하기

> **유의점**
> 1. `VALUES` 절은 사용하지 않는다.
> 2. 추가되는 테이블의 **열 개수**와 서브쿼리의 열 개수가 일치해야 한다.
> 3. 추가되는 테이블의 **자료형**과 서브쿼리의 자료형이 일치해야 한다.

```sql
INSERT INTO EMP_TEMP (EMPNO, ENAME, JOB, MGR, HIREDATE, SAL, COMM, DEPTNO)
SELECT E.EMPNO, E.ENAME, E.JOB, E.MGR, E.HIREDATE, E.SAL, E.COMM, E.DEPTNO
FROM EMP E, SALGRADE S
WHERE E.SAL BETWEEN S.LOSAL AND S.HISAL
  AND S.GRADE = 1;
```

<details>
<summary><b>문제)</b> 이름에 'A'가 들어 있는 EMP 테이블의 데이터를 EMP_TEMP 테이블로 삽입하시오.</summary>

```sql
INSERT INTO EMP_TEMP (EMPNO, ENAME, JOB, MGR, HIREDATE, SAL, COMM, DEPTNO)
SELECT EMPNO, ENAME, JOB, MGR, HIREDATE, SAL, COMM, DEPTNO
FROM EMP
WHERE ENAME LIKE '%A%';
```

</details>

## UPDATE — 데이터 수정

### 형식

```sql
UPDATE 변경할_테이블
SET 열1 = 값1, 열2 = 값2, ..., 열N = 값N
[WHERE 변경 대상 행을 선별하는 조건];
```

```sql
CREATE TABLE DEPT_TEMP2 AS SELECT * FROM DEPT;

-- WHERE를 생략하면 모든 행이 바뀐다 (주의!)
UPDATE DEPT_TEMP2
SET LOC = 'DAEJEON';
```

> ⚠️ `WHERE`를 빼면 **테이블 전체 행**이 수정된다. 꼭 필요한 조건을 함께 쓰자.

<details>
<summary><b>문제)</b> 부서번호가 40인 부서의 이름을 'DATABASE', 지역을 'SEOUL'로 변경하시오.</summary>

```sql
UPDATE DEPT_TEMP2
SET DNAME = 'DATABASE',
    LOC = 'SEOUL'
WHERE DEPTNO = 40;
```

</details>

### 서브쿼리로 데이터 수정

```sql
UPDATE DEPT_TEMP2
SET LOC = 'BUSAN'
WHERE DEPTNO IN (
    SELECT DEPTNO FROM (
        SELECT DEPTNO FROM DEPT_TEMP2 WHERE LOC = 'DAEJEON'
    ) AS T
);
```

> ⚠️ MySQL에서는 **수정 중인 테이블(`DEPT_TEMP2`)을 서브쿼리에서 직접 다시 조회할 수 없다.** 위처럼 서브쿼리를 한 번 더 감싸(`AS T`) 임시 결과로 만들면 해결된다.

## DELETE — 데이터 삭제

### 형식

```sql
DELETE FROM 테이블명
[WHERE 삭제 대상을 선별하는 조건];
```

```sql
CREATE TABLE EMP_TEMP2 AS SELECT * FROM EMP;

DELETE FROM EMP_TEMP2
WHERE JOB = 'MANAGER';
```

> ⚠️ `WHERE`를 빼면 **모든 행이 삭제**된다.

### 서브쿼리로 데이터 삭제

```sql
DELETE FROM EMP_TEMP2
WHERE EMPNO IN (
    SELECT EMPNO FROM (
        SELECT E.EMPNO
        FROM EMP_TEMP2 E, SALGRADE S
        WHERE E.SAL BETWEEN S.LOSAL AND S.HISAL
          AND S.GRADE = 3
          AND E.DEPTNO = 30
    ) AS T
);
```

## 트랜잭션 제어와 세션

### 트랜잭션이란?

**트랜잭션**: 더 이상 나눌 수 없는 **최소 작업 단위**. 계좌 이체처럼 밀접하게 연관된 작업을 하나로 묶은, 하나의 기능을 수행하는 'SQL문 덩어리'다.

- 트랜잭션 안의 여러 명령은 **전부 성공하거나, 전부 취소**된다. → **ALL OR NOTHING**
- 트랜잭션을 제어하는 명령어를 **TCL**(Transaction Control Language)이라고 한다.

| 명령어 | 역할 |
| --- | --- |
| `COMMIT` | 변경 사항을 **영구 확정** |
| `ROLLBACK` | 확정 전 변경 사항을 **취소(되돌리기)** |

```sql
-- 변경 → 잘못됐으면 되돌리기
UPDATE DEPT_TEMP2 SET LOC = 'DAEJEON';
ROLLBACK;   -- 위 UPDATE 취소

-- 변경 → 확정
UPDATE DEPT_TEMP2 SET LOC = 'SEOUL';
COMMIT;     -- 영구 반영 (이후엔 ROLLBACK으로 되돌릴 수 없음)
```

> ⚠️ MySQL은 기본적으로 **자동 커밋(autocommit)** 상태라 명령 하나하나가 바로 확정된다. `ROLLBACK`으로 되돌리려면 먼저 `START TRANSACTION;`(또는 `SET autocommit = 0;`)으로 트랜잭션을 시작해야 한다.
>
> 또한 `CREATE`·`ALTER`·`DROP` 같은 DDL은 실행 시 **자동으로 커밋**되어 `ROLLBACK`으로 되돌릴 수 없다.

### 세션(Session)이란?

**세션**: 데이터베이스에 **접속해서 작업하고 접속을 끊을 때까지의 전체 기간**. (로그인 → 로그아웃까지의 기간과 비슷)

- 세션이 여러 개라는 것은, 현재 데이터베이스에 **사용 중인 연결이 여러 개** 있다는 뜻이다.

### 읽기 일관성

데이터베이스는 여러 곳에서 **동시에** 접근하므로 보통 수많은 세션이 함께 연결된다.

> **읽기 일관성**: 한 세션이 데이터를 변경하는 중일 때, 다른 세션에는 **변경이 확정(COMMIT)되기 전까지** 원래 데이터를 보여주는 특성.

### LOCK (잠금)

> **LOCK**: 한 세션이 조작 중인 데이터는, 트랜잭션이 끝나기 전까지 **다른 세션이 조작할 수 없도록 잠그는 것.** 다른 세션의 접근을 잠시 보류시켜 데이터가 엉키는 것을 막는다.

<details>
<summary><b>문제)</b> EMP_TEMP_PRA 테이블을 EMP와 데이터까지 모두 복제해 만들고, ① 모든 사원 이름을 'PAUL'로 바꾼 뒤 ROLLBACK, ② 모든 SAL을 0으로 바꾼 뒤 COMMIT, ③ 모든 데이터를 삭제하시오.</summary>

```sql
-- 테이블 복제 (구조 + 데이터)
CREATE TABLE EMP_TEMP_PRA AS SELECT * FROM EMP;

-- 트랜잭션 시작 (자동 커밋 해제)
START TRANSACTION;

-- ① 모든 이름을 PAUL로 변경 후 되돌리기
UPDATE EMP_TEMP_PRA SET ENAME = 'PAUL';
ROLLBACK;

-- ② 모든 급여를 0으로 변경 후 확정
UPDATE EMP_TEMP_PRA SET SAL = 0;
COMMIT;

-- ③ 모든 데이터 삭제
DELETE FROM EMP_TEMP_PRA;
```

</details>

## CREATE — 테이블 생성

### 형식

```sql
CREATE TABLE [데이터베이스명.]테이블이름 (
    열1이름 열1자료형,
    열2이름 열2자료형,
    ...
    열N이름 열N자료형
);
```

### 테이블/열 이름 생성 규칙

1. 이름은 **전부 숫자로만** 이루어질 수는 없다. (숫자로 시작은 가능, 한글 가능)
2. 식별자 길이는 **최대 64자**까지.
3. 같은 데이터베이스 안에서 테이블 이름은 **중복될 수 없다.** (한 테이블 안에서 열 이름도 중복 불가)
4. 영문자·숫자(`0~9`)·특수문자 `$`, `_` 를 사용할 수 있다.
5. **SQL 예약어**(SELECT, TABLE 등)는 이름으로 쓸 수 없다.

### 자료형 예시로 테이블 만들기

```sql
CREATE TABLE EMP_DDL (
    EMPNO    INT,
    ENAME    VARCHAR(10),
    JOB      VARCHAR(9),
    MGR      INT,
    HIREDATE DATE,
    SAL      DECIMAL(7, 2),
    COMM     DECIMAL(7, 2),
    DEPTNO   INT
);

DESC EMP_DDL;   -- 테이블 구조 확인
```

> 💡 자주 쓰는 MySQL 자료형: `INT`(정수), `DECIMAL(전체자릿수, 소수자릿수)`(소수), `VARCHAR(길이)`(가변 문자열), `DATE`(날짜)

### 기존 테이블을 복사해 생성하기 (CREATE TABLE AS SELECT)

```sql
-- 구조 + 데이터 전체 복사
CREATE TABLE DEPT_DDL AS SELECT * FROM DEPT;
```

```sql
-- 일부 데이터만 복사
CREATE TABLE EMP_DDL_30 AS SELECT * FROM EMP WHERE DEPTNO = 30;
```

```sql
-- 여러 테이블에서 열 구조만 복사 (WHERE 1<>1 → 데이터 0건)
CREATE TABLE EMPDEPT_DDL AS
SELECT E.EMPNO, E.ENAME, E.JOB, E.MGR, E.HIREDATE, E.SAL, E.COMM,
       D.DEPTNO, D.DNAME, D.LOC
FROM EMP E, DEPT D
WHERE 1 <> 1;
```

## ALTER — 테이블 변경

이미 생성된 테이블의 구조를 변경할 때 사용한다.

```sql
CREATE TABLE EMP_ALTER AS SELECT * FROM EMP;
```

| 작업 | 명령어 |
| --- | --- |
| 열 추가 | `ADD` |
| 열 이름 변경 | `RENAME COLUMN` |
| 열 자료형 변경 | `MODIFY` |
| 열 삭제 | `DROP COLUMN` |

```sql
-- 열 추가
ALTER TABLE EMP_ALTER ADD HP VARCHAR(20);

-- 열 이름 변경 (HP → TEL)
ALTER TABLE EMP_ALTER RENAME COLUMN HP TO TEL;

-- 열 자료형 변경
ALTER TABLE EMP_ALTER MODIFY EMPNO INT;

-- 열 삭제
ALTER TABLE EMP_ALTER DROP COLUMN TEL;
```

### 테이블 삭제 (DROP)

```sql
DROP TABLE EMP_ALTER;
```

## 종합 문제

<details>
<summary><b>문제)</b> 아래 학과(SUBJECT)·교수(PROF) 테이블을 만들고 데이터를 삽입하시오.</summary>

**학과 테이블 — SUBJECT**

| 열 | 이름 | 자료형 |
| --- | --- | --- |
| 학과번호 | SUBNUM | INT |
| 학과명 | SUBNAME | VARCHAR(20) |
| 학과장 번호 | SUBMASTERNUM | INT |

삽입할 데이터 (학과번호 : 학과명 - 학과장 번호)
- 1 : BUSINESS - 10
- 2 : COMPUTER - 20
- 3 : DESIGN - 30

```sql
CREATE TABLE SUBJECT (
    SUBNUM       INT,
    SUBNAME      VARCHAR(20),
    SUBMASTERNUM INT
);

INSERT INTO SUBJECT (SUBNUM, SUBNAME, SUBMASTERNUM) VALUES (1, 'BUSINESS', 10);
INSERT INTO SUBJECT (SUBNUM, SUBNAME, SUBMASTERNUM) VALUES (2, 'COMPUTER', 20);
INSERT INTO SUBJECT (SUBNUM, SUBNAME, SUBMASTERNUM) VALUES (3, 'DESIGN',   30);
```

**교수 테이블 — PROF**

| 열 | 이름 | 자료형 |
| --- | --- | --- |
| 교직원 번호 | PROFNUM | INT |
| 교수명 | PROFNAME | VARCHAR(20) |
| 학과번호 | PROFSUBNUM | INT |
| 연락처 | PROFTEL | VARCHAR(20) |
| 학과장 | PROFMASTERNUM | INT |

조건: 교수 6명 삽입. 학과장은 학과마다 1명씩만 `PROFMASTERNUM`에 10·20·30을 넣고, 나머지 교수는 비워둔다(NULL).

```sql
CREATE TABLE PROF (
    PROFNUM       INT,
    PROFNAME      VARCHAR(20),
    PROFSUBNUM    INT,
    PROFTEL       VARCHAR(20),
    PROFMASTERNUM INT
);

-- 학과장 3명 (PROFMASTERNUM 지정)
INSERT INTO PROF VALUES (1, 'KIM',  1, '010-1111-1111', 10);
INSERT INTO PROF VALUES (2, 'LEE',  2, '010-2222-2222', 20);
INSERT INTO PROF VALUES (3, 'PARK', 3, '010-3333-3333', 30);

-- 일반 교수 3명 (PROFMASTERNUM은 NULL)
INSERT INTO PROF VALUES (4, 'CHOI', 1, '010-4444-4444', NULL);
INSERT INTO PROF VALUES (5, 'JUNG', 2, '010-5555-5555', NULL);
INSERT INTO PROF VALUES (6, 'YOON', 3, '010-6666-6666', NULL);
```

</details>
