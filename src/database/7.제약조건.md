# 제약조건 (Constraint)

> *(SQL 예제는 모두 **MySQL** 기준)*

**제약조건**: 테이블에 저장할 데이터를 제한하는 특수한 규칙. 테이블의 특정 열에 지정하며, 제약 조건에 맞지 않는 데이터는 저장할 수 없다.

| 제약조건 | 의미 |
| --- | --- |
| **NOT NULL** | 지정한 열에 `NULL`을 허용하지 않음 (단, NULL이 아닌 값의 중복은 허용) |
| **UNIQUE** | 지정한 열의 값이 유일해야 함(중복 불가). 단, `NULL`은 중복 검사에서 제외 |
| **PRIMARY KEY** | 유일하면서 `NULL` 불가. 테이블에 **하나만** 지정 가능 |
| **FOREIGN KEY** | 다른 테이블의 열을 참조하여, 그 테이블에 **존재하는 값만** 입력 가능 |
| **CHECK** | 설정한 조건식을 만족하는 데이터만 입력 가능 |

---

## 데이터 무결성

**데이터 무결성**: 데이터베이스에 저장되는 데이터의 **정확성과 일관성**을 보장한다는 의미. 제약조건은 이 무결성을 지키기 위한 안전장치이며, 삽입(INSERT)·수정(UPDATE)·삭제(DELETE) 모든 과정에서 지켜져야 한다.

| 무결성 종류 | 내용 |
| --- | --- |
| **영역 무결성** | 열에 저장되는 값의 적정 여부 확인 (자료형, 형식, NULL 여부 등 정해진 범위 만족) |
| **객체 무결성** | 기본키는 반드시 값을 가져야 하며 `NULL`·중복 불가 |
| **참조 무결성** | 외래키 값은 참조 테이블의 기본키로 존재해야 하며, `NULL`은 가능 |

> 💡 제약조건은 보통 **테이블을 생성하는 시점(CREATE TABLE)** 에 지정한다.

## NOT NULL — 빈 값 금지

```sql
CREATE TABLE TABLE_NOTNULL (
    LOGIN_ID VARCHAR(20) NOT NULL,
    LOGIN_PW VARCHAR(20) NOT NULL
);
```

<details>
<summary><b>문제)</b> ID: 1004 / PW: 1234 를 INSERT 하시오.</summary>

```sql
INSERT INTO TABLE_NOTNULL
VALUES ('1004', '1234');
```

</details>

<details>
<summary><b>문제)</b> TEL VARCHAR(2) 컬럼을 TABLE_NOTNULL 테이블에 추가하시오.</summary>

```sql
ALTER TABLE TABLE_NOTNULL ADD TEL VARCHAR(2);
```

</details>

<details>
<summary><b>문제)</b> TEL 컬럼의 타입을 VARCHAR(20)으로 변경하시오.</summary>

```sql
ALTER TABLE TABLE_NOTNULL MODIFY TEL VARCHAR(20);
```

</details>

<details>
<summary><b>문제)</b> ID가 1004인 데이터의 TEL을 1234로 수정하시오.</summary>

```sql
UPDATE TABLE_NOTNULL
SET TEL = '1234'
WHERE LOGIN_ID = '1004';
```

</details>

## UNIQUE — 중복 금지

```sql
CREATE TABLE TABLE_UNIQUE (
    LOGIN_ID VARCHAR(20) UNIQUE,
    LOGIN_PW VARCHAR(20) NOT NULL,
    TEL VARCHAR(20)
);
```

```sql
-- NULL은 UNIQUE 검사에서 제외되므로 여러 번 넣어도 된다
INSERT INTO TABLE_UNIQUE VALUES (NULL, 'PW', '1234');

-- 일반 값은 유일해야 한다 (같은 값을 또 넣으면 오류)
INSERT INTO TABLE_UNIQUE VALUES ('1004', 'PW', '1234');
```

<details>
<summary><b>문제)</b> ID가 NULL인 데이터를 삭제하시오.</summary>

```sql
DELETE FROM TABLE_UNIQUE
WHERE LOGIN_ID IS NULL;
```

</details>

## PRIMARY KEY (PK) — 유일 + NOT NULL

`PRIMARY KEY`는 **UNIQUE + NOT NULL** 의 특성을 모두 가진다. 즉 중복도, `NULL`도 허용하지 않는다. 주민등록번호나 사원번호처럼 **각 행을 식별**하는 데 사용한다.

```sql
CREATE TABLE TABLE_PK (
    LOGIN_ID VARCHAR(20) PRIMARY KEY,
    LOGIN_PW VARCHAR(20) NOT NULL,
    TEL VARCHAR(20)
);
```

```sql
INSERT INTO TABLE_PK VALUES ('test1', 'pw', '123');  -- 성공
INSERT INTO TABLE_PK VALUES ('test2', 'pw', '456');  -- 성공
INSERT INTO TABLE_PK VALUES (NULL,    'pw', '123');  -- 오류! (PK는 NULL 불가)
```

## FOREIGN KEY (FK) — 다른 테이블 참조

서로 다른 테이블 간의 관계를 정의한다. 한 테이블의 `PRIMARY KEY` 열을, 다른 테이블의 특정 열에서 **참조**하겠다는 의미.

> ℹ️ MySQL(InnoDB)에서는 외래키를 **테이블 레벨**(`CONSTRAINT ... FOREIGN KEY (열) REFERENCES 부모테이블(열)`)로 정의해야 실제로 동작한다.

```sql
-- 부모 테이블 (참조되는 쪽)
CREATE TABLE DEPT_FK (
    DEPTNO INT PRIMARY KEY,
    DNAME  VARCHAR(14),
    LOC    VARCHAR(13)
);
```

```sql
-- 자식 테이블 (참조하는 쪽)
CREATE TABLE EMP_FK (
    EMPNO    INT PRIMARY KEY,
    ENAME    VARCHAR(10),
    JOB      VARCHAR(9),
    MGR      INT,
    HIREDATE DATE,
    SAL      DECIMAL(7, 2),
    COMM     DECIMAL(7, 2),
    DEPTNO   INT,
    CONSTRAINT EMPFK_DEPTNO_FK FOREIGN KEY (DEPTNO) REFERENCES DEPT_FK(DEPTNO)
);
```

<details>
<summary><b>문제)</b> EMP_FK에 다음 데이터를 넣어보시오. (EMPNO 9999, …, DEPTNO 10)</summary>

```sql
-- ❌ 부모 테이블(DEPT_FK)에 DEPTNO 10이 없으면 참조 무결성 위반으로 오류!
INSERT INTO EMP_FK
VALUES (9999, 'TEST_NAME', 'TEST_JOB', NULL, '2001-01-01', 3000, NULL, 10);
```

```sql
-- ✅ 부모 테이블에 먼저 DEPTNO 10을 넣은 뒤 자식 데이터를 넣어야 한다
INSERT INTO DEPT_FK
VALUES (10, 'TEST_NAME', 'TEST_LOC');

INSERT INTO EMP_FK
VALUES (9999, 'TEST_NAME', 'TEST_JOB', NULL, '2001-01-01', 3000, NULL, 10);
```

> 💡 삭제도 반대 순서다. 자식(EMP_FK)이 참조 중인 부모(DEPT_FK) 행은 먼저 삭제할 수 없다.
> ```sql
> DELETE FROM EMP_FK  WHERE DEPTNO = 10;   -- 자식 먼저
> DELETE FROM DEPT_FK WHERE DEPTNO = 10;   -- 그다음 부모
> ```

</details>

## 종합 문제

<details>
<summary><b>문제 1)</b> 학과 테이블(SUBJECT)을 만들고 데이터를 삽입하시오. (학과번호 PK, 모든 열 NOT NULL)</summary>

| 열 | 이름 | 자료형 | 제약 |
| --- | --- | --- | --- |
| 학과번호 | SUBNUM | INT | PK |
| 학과명 | SUBNAME | VARCHAR(20) | NOT NULL |
| 학과장 번호 | SUBMASTERNUM | INT | NOT NULL |

```sql
CREATE TABLE SUBJECT (
    SUBNUM       INT PRIMARY KEY,
    SUBNAME      VARCHAR(20) NOT NULL,
    SUBMASTERNUM INT NOT NULL
);

INSERT INTO SUBJECT VALUES (1, 'BUSINESS', 10);
INSERT INTO SUBJECT VALUES (2, 'COMPUTER', 20);
INSERT INTO SUBJECT VALUES (3, 'DESIGN',   30);
```

</details>

<details>
<summary><b>문제 2)</b> 교수 테이블(PROF)을 만들고 교수 6명을 삽입하시오. (교직원번호 PK, 학과번호 FK→SUBJECT, 학과장 1명씩만 10·20·30, 나머지는 비움)</summary>

| 열 | 이름 | 자료형 | 제약 |
| --- | --- | --- | --- |
| 교직원 번호 | PROFNUM | INT | PK |
| 교수명 | PROFNAME | VARCHAR(20) | NOT NULL |
| 학과번호 | PROFSUBNUM | INT | NOT NULL, FK |
| 연락처 | PROFTEL | VARCHAR(20) | |
| 학과장 | PROFMASTERNUM | INT | |

```sql
CREATE TABLE PROF (
    PROFNUM       INT PRIMARY KEY,
    PROFNAME      VARCHAR(20) NOT NULL,
    PROFSUBNUM    INT NOT NULL,
    PROFTEL       VARCHAR(20),
    PROFMASTERNUM INT,
    CONSTRAINT PROF_PROFSUBNUM_FK FOREIGN KEY (PROFSUBNUM) REFERENCES SUBJECT(SUBNUM)
);

-- 학과장 3명
INSERT INTO PROF VALUES (1, 'KIM',  1, '010-1111-1111', 10);
INSERT INTO PROF VALUES (2, 'LEE',  2, '010-2222-2222', 20);
INSERT INTO PROF VALUES (3, 'PARK', 3, '010-3333-3333', 30);
-- 일반 교수 3명 (학과장 번호는 NULL)
INSERT INTO PROF VALUES (4, 'CHOI', 1, '010-4444-4444', NULL);
INSERT INTO PROF VALUES (5, 'JUNG', 2, '010-5555-5555', NULL);
INSERT INTO PROF VALUES (6, 'YOON', 3, '010-6666-6666', NULL);
```

</details>

<details>
<summary><b>문제 3)</b> 학생 테이블(STU)을 만들고 데이터 10개를 삽입하시오. (학생번호 PK, 전화번호 UNIQUE, 학과번호 FK→SUBJECT, 이름·학과번호 NOT NULL)</summary>
